# 腾讯云部署分步清单（从零到上线）

> 目标：把 GitHub 上的问卷系统（https://github.com/sunking18/MCsurvey.git）部署到腾讯云，通过**已备案域名 + HTTPS** 对外访问，微信扫码不被拦截。
>
> 全程分 6 大步，每步都有可打勾的清单。**备案是最耗时的一步（约 3–20 个工作日），建议第一天就先提交备案，其余步骤并行推进。**

---

## 总览与时间预期

| 阶段 | 内容 | 预计耗时 |
|---|---|---|
| ① 买服务器 | 腾讯云轻量应用服务器 | 10 分钟 |
| ② 买域名 | 在腾讯云注册域名 | 10 分钟 |
| ③ ICP 备案 | 提交材料 → 管局审核 | **3–20 个工作日** ⏳ |
| ④ 部署代码 | 装环境 + 拉代码 + 启动 | 30 分钟 |
| ⑤ Nginx + HTTPS | 反代 + 免费证书 | 20 分钟 |
| ⑥ 上线验证 | 测试 + 出二维码海报 | 15 分钟 |

> 💡 顺序建议：**先做 ①②③**（备案要排队），备案审核期间可以先做 ④（代码先跑起来，用 IP 临时测试），备案通过后再做 ⑤⑥ 正式上线。

---

## ① 购买腾讯云轻量应用服务器

- [ ] 打开 https://cloud.tencent.com/product/lighthouse ，点「立即购买」
- [ ] 地域：选**中国大陆**（如广州 / 上海 / 北京，备案域名必须用大陆节点）
- [ ] 镜像：选 **系统镜像 → Ubuntu 22.04 LTS**（干净系统，后面自己装 Node）
- [ ] 套餐：**2核2G** 起步即可（问卷量不大，最低档够用）
- [ ] 购买时长：按需（1 个月或 1 年）
- [ ] 创建后，在「轻量应用服务器控制台」记下 **公网 IP**（后面处处要用）
- [ ] 设置 root 密码或重置密码（控制台 → 实例 → 更多 → 重置密码）

**登录服务器**（Mac 终端直接用）：
```bash
ssh root@你的公网IP
# 输入刚设置的密码
```

---

## ② 注册域名

- [ ] 打开 https://dns.cloud.tencent.com/ → 域名注册
- [ ] 搜一个好记的域名（如 `mcsurvey.cn`、`xxx-diaocha.com`），`.cn` / `.com` 都行
- [ ] 完成实名认证（个人 / 公司，做心理问卷建议用**公司**主体：河北唠里唠叨心理咨询有限公司）
- [ ] 购买并等待实名审核通过（通常几小时内）

> 域名和服务器建议**都放在腾讯云**，备案时系统能自动关联，最省事。

---

## ③ ICP 备案（最关键、最耗时）⏳

> 微信打开网页时，未备案域名会被拦截提示「风险」。要正式对外发放二维码，**必须备案**。

- [ ] 打开腾讯云备案控制台 https://console.cloud.tencent.com/beian
- [ ] 点「开始备案」，系统检测到你的域名 + 轻量服务器，自动带出信息
- [ ] 填写主体信息：
  - 主体：**河北唠里唠叨心理咨询有限公司**（用营业执照）
  - 网站负责人：可填你本人，需人脸核验
- [ ] 网站信息：网站名称（如「亲子心理问卷」）、网站内容选「其他 / 教育」
- [ ] 上传材料：营业执照、负责人身份证、幕布拍照（腾讯云 App 在线核验，不用邮寄）
- [ ] 提交 → 腾讯云初审（1–2 天）→ 提交管局 → **管局审核（各省 3–20 个工作日）**
- [ ] 收到工信部短信，输入验证码完成备案
- [ ] 备案成功后拿到**备案号**（如 `冀ICP备2026xxxxxx号`），需展示在网站底部

> ⚠️ 备案期间该域名不能对外提供服务。审核期你可以**先用 IP 临时访问测试**（见第④步末尾）。

## ④ 部署代码（备案审核期间即可做，先用 IP 测试）

登录服务器后依次执行：

### 4.1 安装运行环境
```bash
# 更新系统
apt update && apt upgrade -y

# 安装 Node.js 18 LTS
curl -fsSL https://deb.nodesource.com/setup_18.x | bash -
apt install -y nodejs git

# 验证
node -v    # 应显示 v18.x
npm -v
```

- [ ] `node -v` 正常输出

### 4.2 拉取 GitHub 代码
```bash
mkdir -p /var/www && cd /var/www
git clone https://github.com/sunking18/MCsurvey.git survey
cd survey/backend
npm install
```

> 若 clone 私有仓库要求账号密码：用户名 `sunking18`，密码粘贴你的 PAT。公开仓库则无需。

- [ ] 代码已拉到 `/var/www/survey`
- [ ] `npm install` 无报错

### 4.3 配置环境变量
```bash
cd /var/www/survey/backend
cp .env.example .env
nano .env
```
填写：
```env
EXPORT_KEY=设一个复杂的随机字符串（管理员导出数据用）
PORT=3000
```
`Ctrl+O` 保存，`Ctrl+X` 退出。

- [ ] `.env` 已配置，`EXPORT_KEY` 设为强密码

### 4.4 用 pm2 常驻启动
```bash
npm install -g pm2
cd /var/www/survey/backend
pm2 start server.js --name survey
pm2 save
pm2 startup    # 按提示复制执行它输出的那行命令，实现开机自启
```

- [ ] `pm2 list` 显示 survey 状态为 online

### 4.5 开放安全组端口（临时用 IP 测试）
- [ ] 轻量控制台 → 防火墙 → 添加规则，放行 **3000**（临时测试）、**80**、**443**
- [ ] 浏览器访问 `http://你的公网IP:3000/index.html`，能看到问卷说明页 ✅

> 备案通过前只能这样用 IP + 端口临时自测；正式对外发二维码要等备案 + 域名 + HTTPS。

## ⑤ 配置 Nginx 反向代理 + HTTPS（备案通过后）

### 5.1 域名解析到服务器
- [ ] 腾讯云 DNS 控制台 → 你的域名 → 添加记录：
  - 主机记录 `@`（或 `survey`），记录类型 `A`，记录值填**公网 IP**
- [ ] 等几分钟生效，`ping 你的域名` 能解析到该 IP

### 5.2 安装 Nginx
```bash
apt install -y nginx
```

### 5.3 写站点配置
```bash
nano /etc/nginx/sites-available/survey
```
粘贴（把 `survey.你的域名.cn` 换成实际域名）：
```nginx
server {
    listen 80;
    server_name survey.你的域名.cn;

    location / {
        proxy_pass http://127.0.0.1:3000;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```
> ⚠️ `X-Forwarded-For` 这行必须有，否则后端记录到的 IP 全是 `127.0.0.1`。

启用并重载：
```bash
ln -s /etc/nginx/sites-available/survey /etc/nginx/sites-enabled/
nginx -t          # 测试配置无误
systemctl reload nginx
```

- [ ] `http://survey.你的域名.cn/index.html` 能打开

### 5.4 一键上 HTTPS（Let's Encrypt 免费证书）
```bash
apt install -y certbot python3-certbot-nginx
certbot --nginx -d survey.你的域名.cn
# 按提示输入邮箱、同意条款，选择自动跳转 HTTPS
```
Certbot 会自动改写 Nginx 配置并配好证书，还会自动续期。

- [ ] `https://survey.你的域名.cn/index.html` 打开有小锁 🔒

### 5.5 关掉临时的 3000 端口
- [ ] 备案 + HTTPS 都好后，回安全组**删除 3000 端口规则**，只留 80/443，更安全。

## ⑥ 上线验证 + 出二维码海报

- [ ] 浏览器访问 `https://survey.你的域名.cn/index.html`，走完整流程：说明页 → 基础信息（身份字段必填）→ 答 35 题 → 看个人报告
- [ ] 手机微信扫一扫打开同一地址，确认**不再有风险拦截提示**
- [ ] 打开 `https://survey.你的域名.cn/poster.html`，在输入框填入首页地址 → 自动生成二维码 → 点「下载 PNG 海报」
- [ ] 打开 `https://survey.你的域名.cn/admin.html`，输入 `EXPORT_KEY` → 下载 CSV，检查 `ip / device_model / 答题时长(秒) / lie_flag` 等字段是否正确（此时 IP 应为真实公网 IP，不再是 127.0.0.1）
- [ ] 在网站底部/海报上标注**备案号**（合规要求）

## 后续更新代码怎么办？

以后本地改了代码、推到 GitHub 后，服务器上只需：
```bash
cd /var/www/survey
git pull
cd backend && npm install     # 如果依赖有变化
pm2 restart survey
```

## 一句话流程回顾

买机 → 买域名 → **先提交备案（等审核）** → 审核期间用 IP 把代码跑起来自测 → 备案通过 → 域名解析 + Nginx + HTTPS → 关临时端口 → 出二维码海报，正式上线。
