# 登录服务器并上传（部署）代码步骤

> 适用于：已买好腾讯云轻量服务器、域名备案已通过、已用 HTTPS（Nginx）反代的场景。  
> 目标：把 GitHub 仓库 `https://github.com/sunking18/MCsurvey.git` 拉到服务器并跑起来。

---

## 一、拿到服务器登录信息

1. 登录 [腾讯云控制台](https://console.cloud.tencent.com/) → 轻量应用服务器 → 实例详情。
2. 找到：
   - **公网 IP**（例如 `106.x.x.x`）
   - **SSH 端口**（默认 22，若改过请用改后的端口）
   - 用户名：Ubuntu 默认 `ubuntu`；CentOS 默认 `root`；这里按 `ubuntu` 为例。
   - 登录方式：推荐「密钥对」或「密码」。

---

## 二、从本机登录服务器（SSH）

### Mac / Linux

打开终端，运行：

```bash
ssh -i ~/.ssh/你的私钥.pem ubuntu@106.x.x.x
```

如果用密码：

```bash
ssh ubuntu@106.x.x.x
# 然后输入密码
```

> 第一次登录会提示 `Are you sure you want to continue connecting?` 输入 `yes` 回车。

### Windows

- **推荐 1**：安装 Git Bash 或 Windows Terminal，然后像 Mac 一样用 `ssh`。
- **推荐 2**：用 [PuTTY](https://www.putty.org/)。
- 用户名填 `ubuntu`，IP 填公网 IP，端口 22。

## 三、进入服务器后安装依赖

```bash
# 1. 更新系统
sudo apt update && sudo apt upgrade -y

# 2. 安装 Node.js 22（推荐，与本地一致）
# 官方脚本安装
sudo apt install -y curl

# 下载 NodeSource 22.x 安装脚本

# 安装 curl 若未安装
# 添加 NodeSource 源并安装 Node 22
curl -fsSL https://deb.nodesource.com/setup_22.x | sudo -E bash -
sudo apt-get install -y nodejs

# 验证
node -v   # 应显示 v22.x.x
npm -v    # 应显示 10.x.x

# 3. 安装 git
sudo apt install -y git

# 4. 安装 pm2（让服务常驻后台）
sudo npm install -g pm2
```

## 四、从 GitHub 拉取代码

```bash
# 1. 选一个目录放项目（推荐 /var/www 或 /home/ubuntu）
cd /home/ubuntu

# 2. 克隆仓库（只拉一次）
git clone https://github.com/sunking18/MCsurvey.git

# 3. 进入项目
cd MCsurvey

# 4. 安装后端依赖
npm install
# 或如果 node_modules 已上传想只补依赖
npm install --production
```

> 如果后续代码有更新，重复下面「更新代码」部分即可。

## 五、创建服务器上的 .env

**`.env` 不会随 git 上传，必须在服务器手动创建。**

```bash
cd /home/ubuntu/MCsurvey/backend

# 复制模板
cp .env.example .env

# 编辑真实值
nano .env
```

填入：

```ini
# 数据导出密钥（给 /admin.html 用，生产环境务必改强密码）
EXPORT_KEY=你自定义的强密码

# 服务端口
PORT=3000

# 微信公众号网页授权（已配置）
MP_APPID=wx1021ba900fb43d2f
MP_SECRET=你的真实AppSecret

# 部署后的站点根域名（保险）
# 例如：PUBLIC_BASE_URL=https://survey.你的域名.com
PUBLIC_BASE_URL=
```

按 `Ctrl+X` → `Y` → 回车 保存。

## 六、启动后端服务（pm2 常驻）

```bash
cd /home/ubuntu/MCsurvey/backend

# 用 pm2 启动，命名为 survey
pm2 start server.js --name survey

# 保存 pm2 进程列表，重启服务器后自动启动
pm2 save
pm2 startup systemd
# 这条命令会提示你运行一条 sudo 命令，复制粘贴执行即可
```

检查运行状态：

```bash
pm2 status
pm2 logs survey
```

## 八、配置 Nginx（HTTPS 反代）

> 必须先完成域名 ICP 备案，并已在腾讯云轻量服务器防火墙开放 80/443/22 端口。

安装 Nginx 和 certbot：

```bash
sudo apt install -y nginx certbot python3-certbot-nginx
```

创建站点配置文件：

```bash
sudo nano /etc/nginx/sites-available/MCsurvey
```

内容如下（把 `survey.你的域名.com` 换成真实域名）：

```nginx
server {
    listen 80;
    server_name survey.你的域名.com;

    location / {
        proxy_pass http://127.0.0.1:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_cache_bypass $http_upgrade;
    }
}
```

启用配置：

```bash
sudo ln -s /etc/nginx/sites-available/MCsurvey /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl restart nginx
```

申请 HTTPS 证书：

```bash
sudo certbot --nginx -d survey.你的域名.com
# 按提示操作，选自动重定向 HTTP → HTTPS
```

## 九、公众号后台配置「网页授权域名」

1. 登录 [微信公众号后台](https://mp.weixin.qq.com/) → 设置与开发 → 公众号设置 → 功能设置 → 网页授权域名。
2. 点击「设置」，填入：`survey.你的域名.com`（不加 `https://`）。
3. 点击下载 `MP_verify_b1s8XUY2r8hgdUhB.txt`（已放在仓库根目录，随 git pull 一起到 /home/ubuntu/MCsurvey/ 下）。
4. 确保通过 Nginx 可以访问：

```
https://survey.你的域名.com/MP_verify_b1s8XUY2r8hgdUhB.txt
```

看到文件内容即成功，再在微信后台保存网页授权域名。

## 常见问题速查

| 现象 | 原因 | 处理 |
|---|---|---|
| 本地 `curl 127.0.0.1:3000` 不通 | 后端没启动 / 防火墙 | `pm2 logs survey` 看错误；检查安全组是否放行 3000 |
| 域名打不开 80/443 | 防火墙/安全组未放行 | 腾讯云控制台 → 防火墙 → 添加 80/443/22 入站规则 |
| 微信授权回调失败 | redirect_uri 不是 https 或域名不匹配 | 检查 `X-Forwarded-Proto`；必要时在 `.env` 设 `PUBLIC_BASE_URL` |
| MP_verify 文件 404 | 文件不在 /home/ubuntu/MCsurvey/ 根下或 Nginx 没反代 | `ls /home/ubuntu/MCsurvey/MP_verify_*` 确认 |
| 服务器重启后服务没起来 | pm2 自启动没配 | `pm2 startup systemd` 按提示执行 |

---

## 一句话回顾

> `ssh ubuntu@IP` → `git clone` 或 `git pull` → 写 `backend/.env` → `pm2 start backend/server.js --name survey` → 配 Nginx + HTTPS → 公众号后台填网页授权域名 → 海报二维码用 HTTPS 域名生成。
