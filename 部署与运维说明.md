# 问卷部署操作清单（纯匿名采集，无需微信登录）

> 本方案已简化为：**扫码即填、人人留痕**。不需要任何微信资质或登录授权，任何人访问部署地址即可作答；系统自动记录设备型号、IP、答题时间等留痕信息。

---

## 一、系统自动留痕的字段

用户作答时无需手动填写以下信息，全部由前端采集 + 后端记录：

| 字段 | 来源 | 说明 |
|---|---|---|
| `ip` | 后端从请求头读取 | 客户端 IP；公网部署并经 Nginx 反代设置 `X-Forwarded-For` 后即为真实公网 IP |
| `device_model` | 前端从 UA 解析 | 粗略设备分类：`iPhone` / `iPad` / `Android` / `Mac` / `Windows` / `Linux` / `PC` |
| `user_agent` | 前端传 + 后端兜底 | 浏览器完整 UA 串，可用于进一步分析具体机型 / 浏览器 |
| `screen` | 前端采集 | 屏幕分辨率，如 `390x844` |
| `start_time` | 前端记录 | 点击「开始作答」的时刻（ISO8601） |
| `submit_time` | 前端记录 | 点击「提交」的时刻（ISO8601） |
| `server_time` | 后端记录 | 服务端收到提交的时刻 |
| `答题时长(秒)` | 导出时计算 | `submit_time − start_time`，用于识别异常快速作答 |

> 关于「手机型号」精度：受微信/浏览器 UA 限制，iPhone 各型号 UA 几乎一致，只能识别到大类 `iPhone`；Android 可识别到具体品牌机型（如 `SM-G991B`）。如需精确到型号，需引入 UA 解析库或微信 JS-SDK，意义不大，当前方案足够区分设备类型与平台。

## 二、前置条件

1. 一台可公网访问的 Linux 服务器（推荐阿里云 / 腾讯云 / 华为云），需固定公网 IP。
2. 一个**已 ICP 备案的域名** + HTTPS 证书（Let's Encrypt 免费）。
3. Node.js 18+ 运行环境。
4. （可选）pm2 用于生产持久化。

> 注意：微信扫描二维码打开网页，本身不要求备案域名——但微信对「非微信官方网页」会有安全提示，且若要正式对外传播，域名建议完成 ICP 备案并配置 HTTPS，否则用户打开时会被微信拦截或提示风险。

## 三、服务器端部署

### 1. 上传后端并安装依赖

```bash
cd /var/www/survey/backend
npm install        # 依赖：express、cors、sqlite3
```

### 2. 创建 `.env` 文件

```bash
cp .env.example .env
```

编辑 `.env`：

```env
# 数据导出密钥（管理员在 /admin.html 输入，请设置为强密码）
EXPORT_KEY=请设置一个足够复杂的随机字符串

# 端口，默认 3000
PORT=3000
```

### 3. 启动服务（生产建议用 pm2）

临时启动测试：

```bash
node server.js
```

生产持久化（推荐 pm2）：

```bash
npm install -g pm2
pm2 start server.js --name survey-backend
pm2 save
pm2 startup
```

## 四、Nginx 反向代理 + HTTPS

后端跑在 `127.0.0.1:3000`，Nginx 配置示例（**务必带上 `X-Forwarded-For`，否则后端读不到真实 IP**）：

```nginx
server {
    listen 80;
    server_name survey.example.com;
    return 301 https://$server_name$request_uri;
}

server {
    listen 443 ssl http2;
    server_name survey.example.com;

    ssl_certificate /etc/letsencrypt/live/survey.example.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/survey.example.com/privkey.pem;

    location / {
        proxy_pass http://127.0.0.1:3000;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

获取证书（Let's Encrypt）：

```bash
sudo certbot --nginx -d survey.example.com
```

## 五-B、启用微信网页授权（扫码即填 + 获取微信信息 + 再看报告）

> 适用场景：希望用户**用微信扫海报二维码 → 在微信内自动授权 → 直接作答**，并获取微信昵称/头像/性别/城市，且同一微信用户**再次打开可回看历史报告**。需在公众号后台配置网页授权域名。

### 1. 配置前提（必须满足）

- 一个**已认证的服务号/订阅号**，在「设置与开发 → 基本配置」拿到 **AppID** 和 **AppSecret**。
- 部署域名**已完成 ICP 备案 + HTTPS**（见本文档第四、五节）。
- 在公众号后台「设置与开发 → 公众号设置 → 功能设置 → 网页授权域名」：
  - 填写你的部署域名（如 `survey.example.com`，**不要带 http/https**）；
  - 下载微信下发的 `MP_verify_xxxx.txt` 校验文件，放到网站根目录，确保 `https://survey.example.com/MP_verify_xxxx.txt` 可公网访问。

### 2. 后端配置

编辑 `backend/.env`，填入：

```env
MP_APPID=你的公众号AppID
MP_SECRET=你的公众号AppSecret
```

重启服务（pm2 restart survey）。`/api/health` 返回里 `mp_enabled` 应为 `true`。

### 3. 用户体验流程

1. 微信扫海报二维码 → 在微信内打开首页 → 点「开始作答」。
2. 首次：自动跳转微信授权页（snsapi_userinfo，弹一次授权）→ 返回首页，顶部显示「✅ 已通过微信授权：昵称」。
3. 填写基础信息（微信用户同样需填姓名缩写/手机后四位/城市/社区厂区）→ 作答 35 题 → 看报告。
4. **该微信用户再次扫二维码打开**：顶部显示「您之前已完成 N 次作答，可随时回看报告」+「查看最近一次报告」按钮 → 直接看回历史报告（报告与首次作答完全一致，因用同一份提交数据重算）。

### 4. 说明

- 微信授权仅在**微信内置浏览器**内生效；非微信环境（普通浏览器）自动退化为匿名采集。
- 获取的微信信息：昵称、头像、性别、城市/省份/国家；**不含手机号、不含实名**。
- 识别回头用户依赖微信 `openid`（同一用户同一公众号稳定不变），与设备/IP 无关。
- 历史报告通过 `openid` 关联，允许用户反复作答，每次均独立留档（保留历史）。

## 六、测试流程（无需微信）

1. 浏览器访问 `https://survey.example.com/index.html`，确认显示「问卷说明」页。
2. 点击「开始作答」，确认进入基础信息页，且身份识别字段（姓名缩写 / 手机后四位 / 城市 / 社区厂区）均为必填。
3. 用微信扫描海报二维码，确认在微信内打开同一页面。
4. 完整填写问卷并提交，确认看到个人报告。
5. 访问 `https://survey.example.com/admin.html`，输入导出密钥，下载 CSV / JSON 数据。
6. 检查导出数据中 `ip`、`device_model`、`user_agent`、`start_time`、`submit_time`、`答题时长(秒)`、`lie_flag` 等字段是否正确。

## 七、常见问题

| 问题 | 原因 | 处理 |
|---|---|---|
| 导出 CSV 中 `ip` 显示 `::1` 或 `127.0.0.1` | 本地 / 内网访问，或 Nginx 未传 `X-Forwarded-For` | 公网部署并确保 Nginx 配置含 `proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;` |
| `device_model` 只到 `iPhone` 大类，不到具体型号 | 微信 / 浏览器 UA 限制 | 属正常；如需更细可引入 UA 解析库（意义有限） |
| `答题时长(秒)` 为空或异常 | 设备时区 / 时间格式问题 | 已统一使用 ISO8601，正常网络环境下稳定 |
| 微信内打开提示「非微信官方网页」风险 | 域名未备案或未 HTTPS | 完成 ICP 备案并配置 HTTPS |
| 提交后报告正常但 admin 下载为空 | 密钥错误 | 确认 `.env` 中 `EXPORT_KEY` 与 admin 页输入一致 |
| 微信内授权后首页未显示「已授权/再看报告」 | 网页授权域名未配置或 HTTPS 异常 | 检查公众号后台网页授权域名、校验文件可访问、证书有效；`/api/health` 的 `mp_enabled` 应为 true |
| 「再看报告」只在微信内可用，外部浏览器打不开 | 设计如此（依赖 openid） | 该能力基于微信 openid 识别，必须在微信内重新授权后回看 |

## 八、安全与隐私建议

- `EXPORT_KEY` 请设置为足够复杂的随机字符串，不要泄露。
- 数据库文件 `backend/data/responses.db` 请定期备份，并限制服务器文件访问权限（不要放到 web 根目录可下载的位置）。
- 收集 IP、设备信息属于个人信息，建议在首页或海报中增加简短的《隐私说明 / 知情同意》文字，提升合规性与用户信任。
- 若仅需内部使用，可将服务器放在企业内网 / VPN 环境，仅授权人员可访问。
